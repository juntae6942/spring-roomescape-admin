## v2

   2-1. AI에게 지침으로 설명 할 때, 공식문서 링크 알려주는 프롬프트를 추가한다.
3. 피드백 루프를 사용해서 고민하는 시간이 얼마나 필요한지 반복해서 시도해보고, 적정 시간을 파악 해보는 방법을 적용하면 좋을 것 같다.
4. AI와 토론을 통해서 정리한다.  
## v3
1. 블로그글, 유튜브, 책, 다른 사람의 인사이트를 통해 우선 학습한다.
2. 자료가 부족한 경우 AI에게 공식문서 링크를 받아서 읽어야하는 공식문서 부분을 찾아서 읽어본다.

   2-1. AI에게 지침으로 설명 할 때, 공식문서 링크 알려주는 프롬프트를 추가한다.

3. 피드백 루프를 사용해서 고민하는 시간이 얼마나 필요한지 반복해서 시도해보고, 적정 시간을 파악 해보는 방법을 적용하면 좋을 것 같다.
4. AI와 토론을 통해서 정리한다.
5. '인지적 마찰'을 의도적으로 설계하라
- AI를 사용할 때, 하나의 채팅창에서 모든 것을 해결하지 말고, 두 개의 완전히 다른 역할을 가진 채팅방(또는 챗봇 인스턴스)을 생성한다.
- Chat A (내비게이터 역할 - 학습 시작 전용):
  초기 설정(System Prompt): "너는 링크 제공자야. 내가 어떤 기술이나 개념에 대해 물어보면, 절대 그 개념에 대해 설명하지 마. 오직 공식 문서의 정확한 URL 링크와, 공식 문서 내에서 검색해야 할 키워드만 딱 2줄로 대답해 줘."
- Chat B (페어 프로그래머 역할 - 학습 후 토론 전용): 공식 문서를 읽고, 예제 코드를 직접 쳐보고, 충분한 고민(3단계)을 거친 후에만 이 채팅방을 연다. 여기서 본인이 이해한 바를 설명하고, AI와 심도 있는 토론을 진행한다.
